```typescript

```

#### CORS (교차 출처 리소스 공유)
```typescript
app.enableCors({
  origin: process.env.CORS_ORIGIN || ['http://localhost:3000'],
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization', 'Accept'],

```

#### Rate Limiting (3단계 정책)
```typescript
ThrottlerModule.forRootAsync({
  useFactory: () => [
    { name: 'short', ttl: 60000, limit: 10 },   // 1분당 10회
    { name: 'medium', ttl: 60000, limit: 30 },  // 1분당 30회
    { name: 'long', ttl: 60000, limit: 100 },   // 1분당 100회
  ],
})
```

#### Compression (응답 압축)
```typescript
compression({
  threshold: 1024,  // 1KB 이상만 압축
})
```

#### Validation Pipe (자동 검증)
```typescript
new ValidationPipe({
  whitelist: true,              // 미정의 속성 제거
  forbidNonWhitelisted: true,   // 미정의 속성 에러
  transform: true,              // DTO 타입 변환
})
```

### 4. 인증/권한 시스템 (완성) ✅

#### JWT 인증
- **JwtStrategy**: Access Token 검증
- **JwtRefreshStrategy**: Refresh Token 검증
- **JwtAuthGuard**: 인증 가드 (@Public 지원)
- **JwtRefreshAuthGuard**: Refresh 가드

#### RBAC (역할 기반 접근 제어)
- **RolesGuard**: 역할 검증 가드
- **@Roles()**: 역할 데코레이터
- **Role enum**: USER, SELLER, ADMIN

#### 개발자 친화 데코레이터
- **@Public()**: 공개 API 표시
- **@Roles(...roles)**: 역할 기반 보호
- **@CurrentUser()**: 사용자 정보 추출

## 🔑 핵심 기술 패턴 (확립됨)

### 1. Repository 패턴
```typescript
@Injectable()
export class UsersRepository {
  constructor(private prisma: PrismaService) {}

  async findById(id: string) {
    return this.prisma.user.findUnique({
      where: { id },
      include: { /* 관계 로드 */ },
    });
  }

  async findMany(query: QueryDto) {
    const { page = 1, limit = 20 } = query;
    const skip = (page - 1) * limit;

    const [items, total] = await Promise.all([
      this.prisma.user.findMany({ skip, take: limit }),
      this.prisma.user.count(),
    ]);

    return { items, total, page, limit };
  }
}
```

### 2. Service 패턴
```typescript
@Injectable()
export class UsersService {
  constructor(private repository: UsersRepository) {}

  async create(dto: CreateUserDto) {
    // 비즈니스 로직
    // 권한 검증
    return this.repository.create(dto);
  }
}
```

### 3. Controller 패턴
```typescript
@Controller('users')
@UseGuards(JwtAuthGuard)
export class UsersController {
  constructor(private service: UsersService) {}

  @Post()
  @Public()
  async create(@Body() dto: CreateUserDto) {
    return this.service.create(dto);
  }

  @Get('profile')
  async getProfile(@CurrentUser('userId') userId: string) {
    return this.service.findById(userId);
  }
}
```

### 4. 상태 머신 패턴 (Orders)
```typescript
// 주문 상태 전환
PENDING → PAYMENT_PENDING → PAID → SHIPPING → DELIVERED → CONFIRMED
                                ↓
                            CANCELLED

// 검증 로직
async updateStatus(orderId: string, newStatus: OrderStatus) {
  const order = await this.repository.findById(orderId);

  const allowedTransitions = {
    [OrderStatus.PENDING]: [OrderStatus.PAYMENT_PENDING, OrderStatus.CANCELLED],
    [OrderStatus.PAYMENT_PENDING]: [OrderStatus.PAID, OrderStatus.CANCELLED],
    // ... 나머지 전환 규칙
  };

  if (!allowedTransitions[order.status].includes(newStatus)) {
    throw new BadRequestException('허용되지 않은 상태 전환입니다');
  }

  return this.repository.updateStatus(orderId, newStatus);
}
```
